# Design MD: Dashboard Workorder Analytics

> 来源：`数据可视化2.vue`

## 1. 页面定位

这是一个标准的数据分析型 dashboard，用于展示工单总量、处理状态、评分、项目归属、热门案例和热搜关键词等统计信息。

它不是大屏炫技页，而是后台工作台中的“高频监控视图”。

## 2. 视觉基调

- 白色主画布 + 卡片式信息模块
- 图表主题色由 `--accent-main-color` 驱动
- 通过主色渐变带生成图表调色盘，保证不同图表风格统一
- 强调“业务信息密度 + 可扫读性”，弱化装饰噪音

## 3. 布局结构

### 整体网格

- 双栏布局：`3fr / 6fr`
- 间距统一为 `10px`
- 页面主高度约 `85vh`

### 左侧栏

- 四段纵向结构
- 上方：3 个小指标卡
- 中上：工单评分卡，内部再拆成平均评分与评分分布
- 中下：工单趋势统计
- 下方：工单状态统计

### 右侧主区

- 三列卡片网格
- 第一行整行跨列：工单归属项目统计
- 第二行三块并列：热门案例、热搜问题分类、热搜关键字

## 4. 核心组件

### 指标卡

- 内容：提单数 / 处理中数 / 工单完成数
- 数字居中，大字号 `28px`，粗体
- 使用动画数字组件 `count-to`

### 图表卡

- 卡头左侧有 4px 主题色竖条
- 卡片背景白色，`1px` 浅边框，`6px` 圆角
- 阴影轻量：`0 2px 12px rgba(0, 0, 0, 0.06)`

### 图表类型

- 仪表盘：平均评分
- 柱状 / 分布图：评分分布、项目统计
- 折线 / 趋势图：工单趋势
- 状态统计图：工单状态
- 排名列表：热门案例、热搜问题分类
- 词云：热搜关键字

### 空状态

- 所有图表都需要 `el-empty`
- 空态要垂直水平居中
- 不允许图表容器塌陷

## 5. 交互规则

- 父组件传入筛选条件与统计数据，当前组件负责渲染，不主动重复请求
- 数字指标切换数据后要重新播放动画
- 柱图支持 `dataZoom`
- 项目统计支持自动 hover 轮播，并在鼠标移入时暂停
- 图表无数据时销毁旧实例，避免展示脏状态

## 6. 色彩与数据可视化规范

- 主色来自业务主题色
- 通过主色到白色、白色到副色生成渐变色带
- 同一页面中图表颜色必须从统一色带分配，不要每个图表各自配色
- 次级文字、坐标轴、弱提示统一使用灰阶

## 7. 业务表达重点

- 数量类指标优先放左侧，便于快速扫读
- 质量类指标放在评分模块
- 归属、热度、关键词放右侧作为横向扩展信息
- 页面结构体现“先总览，再原因，再热点”

## 8. 设计输出建议

- Figma 中建议拆成以下 frame：
  - 指标与评分区
  - 趋势与状态区
  - 项目统计区
  - 热门案例 / 热搜分类 / 热词区
- 需要单独沉淀一页图表规范：
  - 卡片头
  - 数字指标样式
  - 空态样式
  - 渐变色带规则

## 9. AI 生成提示

> 设计一个企业后台工单统计 dashboard，白底卡片布局，桌面端优先，左窄右宽。左侧放 3 个核心数字卡、评分模块、趋势图和状态图；右侧放项目统计、热门案例排行、热搜问题分类和关键词词云。图表统一使用主题色衍生渐变带，风格克制、专业、信息密集，不要做炫技型大屏。
