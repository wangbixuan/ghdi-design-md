# Design MD: Detail Config Page

> 来源：`详情页.vue`

## 1. 页面定位

这是一个“详情 + 配置 + 编辑”融合页，以通知公告编辑为例，承担：

- 模板选择
- 发布方式设置
- 规则绑定
- 正文编辑
- 附件上传
- 预览
- 编辑态规则变更提醒

它不是纯只读详情页，更接近“业务对象配置详情页”。

## 2. 视觉基调

- 仍然维持企业后台白底体系
- 通过双列信息表单 + 大编辑区建立层次
- 重视文档编辑感，而不是传统表单堆叠感
- 辅助浮层、抽屉、预览框形成多层工作流

## 3. 布局结构

### 页面头

- 两行居中标题
- 第一行是组织名称
- 第二行是页面动作态：新增通知公告 / 编辑通知公告

### 主表单区

- 顶部使用两列网格布局
- 标签宽度 `120px`
- 列间距 `24px`
- 宽屏双列，窄屏单列

### 内容区

- 公告内容独立成整行编辑区
- 下方分别是附件区、封面图片区
- 底部为保存 / 取消操作区

### 辅助层

- 预览弹窗
- 新增规则抽屉
- 规则详情抽屉
- 右下角规则变更悬浮提示

## 4. 核心组件

### 顶部配置表单

- 模板选择
- 重要程度
- 发布时间方式
- 定时发布时间
- 发文规则
- 权重排序
- 公告标题

### 富文本编辑器

- 采用 `TemplateEditor`
- 支持变量填值
- 支持预览前保存
- 编辑区高度至少 `400px`

### 上传区

- 通知公告附件
- 封面图片
- 布局与主编辑区左边缘对齐
- 为右侧变量面板预留空间

### 规则提示条

- 固定右下角悬浮
- 白底浅边框
- 阴影比普通卡片更强
- 提供“同步覆盖（推荐）”和“暂不同步”两个动作

## 5. 交互规则

- 新增态与编辑态共用同一页面壳层
- 定时发布时必须校验未来时间
- 模板切换后要同步富文本结构和绑定规则
- 编辑态如果规则快照与当前规则不一致，显示悬浮提醒
- 预览不是纯前端预览，而是先保存获取实例 ID，再展示正式预览
- 抽屉用于承载规则创建与规则详情，避免离开当前编辑上下文

## 6. 视觉与可用性细节

- 页面外边距：`16px 24px 24px`
- 标题字号约 `19px`，字重 600
- 富文本区域和附件区域应左右对齐，保持文档感
- 上传按钮使用红色强调，表示“工具动作”而非主流程动作
- 表单底部按钮居中排布

## 7. 业务约束

- 发布方式与规则快照强绑定
- 模板不仅决定内容骨架，也可能决定绑定规则
- 预览依赖后端持久化 ID
- 编辑页需要处理“当前内容”和“最新规则”之间的偏差

## 8. 适合抽象的设计模式

这类页面可以沉淀为通用“配置详情页模板”：

- 双列设置区
- 文档编辑主区
- 附件与封面扩展区
- 预览能力
- 规则或依赖项的悬浮提醒
- 抽屉承载关联对象操作
