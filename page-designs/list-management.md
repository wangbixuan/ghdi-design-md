# Design MD: List Management Page

> 来源：`列表页.vue`

## 1. 页面定位

这是典型的企业后台列表管理页，以“资产登记一览表”为例，承担：

- 数据查询
- 行内操作
- 批量选择
- 新增 / 编辑 / 删除
- 导入
- 打印
- 查看详情

这类页面是后台系统最高频的基础骨架之一。

## 2. 视觉基调

- 表格优先，不抢视觉焦点
- 操作文本化，弱化大按钮堆叠
- 主色统一使用 `var(--theme-color)`
- 弹窗作为辅助操作面板，不应抢夺列表主层级

## 3. 布局结构

### 主体结构

- 顶部：表格工具栏，由 `baseTableV2` 承载
- 中部：数据表格
- 隐含层：多个对话框用于详情、导入、打印、二维码、公用资产确认

### 工具栏动作

- 登记
- 导入
- 打印

### 行内动作

- 数据确认
- 查看
- 更多下拉：
  - 修改
  - 删除
  - 二维码
  - 是否公用资产

## 4. 核心组件

### 基础表格容器

- 使用 `baseTableV2`
- 搜索、分页、批量选择由底层容器处理
- 页面只补充动作列和工具栏 slot

### 行内操作模式

- 一级高频动作使用文本按钮
- 次级动作收纳进 `el-dropdown`
- 适合高密列表，避免每行过多实体按钮

### 弹窗体系

- 详情弹窗：宽度 `90%`
- 导入弹窗：全屏
- 打印弹窗：宽度 `50%`
- 二维码 / 公用资产确认：宽度 `30%`

设计上属于“列表主页面 + 任务弹层”的典型后台模式。

## 5. 交互规则

- 列表加载时先取权限，再决定可见操作
- “数据确认”是前置动作，确认后才允许查看二维码等后续能力
- 批量勾选后可以统一打印标签
- 删除、确认等高风险动作必须二次确认
- 导入完成后刷新表格
- 关闭弹窗后回到列表上下文，不应跳页

## 6. 视觉细节规范

- 行内文本操作使用主题色，字重 600
- 下拉入口文案“更多”，后接箭头图标
- 弹窗底部按钮宽度统一，常见值 `150px`
- 二维码弹窗内容居中
- 批量打印预览应强调标签内容而非原始表格

## 7. 业务约束

- 权限影响操作可见性
- 已确认与未确认数据的操作集不同
- 公用资产是状态确认型弹窗，不是表单页
- 导入模板是固定文件地址，说明该类列表页经常和模板化数据流转绑定

## 8. 适用的设计模式总结

这类页面适合抽象成统一模板：

- 顶部搜索 + 工具栏
- 中部表格
- 行内主操作 + 更多菜单
- 弹窗完成扩展任务
- 权限与状态共同控制按钮显隐
