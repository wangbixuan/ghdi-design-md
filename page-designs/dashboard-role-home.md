# Design MD: Dashboard Role Home

> 来源：`数据可视化1.vue`

## 1. 页面定位

这是一个角色分发型首页，不是单一信息面板，而是根据用户角色切换到不同工作台：

- 超级管理员：进入 `systemAdmin`
- 部门管理员 / 部门负责人：进入 `departmentAdmin`
- 普通人员：进入 `ordinaryStaff`
- 当前文件内直接承载的公共视图是“资产管理员”配置总览表

设计上，这类页面的核心不是视觉冲击，而是“登录后立刻进入正确职责场景”。

## 2. 视觉基调

- 白底后台界面，低装饰
- 重点依赖表格、标题、留白分区建立秩序
- 字体以 14px 业务正文字号为主
- 标题使用深灰 `#3D3D3D`
- 容器背景纯白，外层轻留白，不叠加复杂阴影

## 3. 布局结构

### 页面骨架

- 顶层根据角色直接切换不同子页面
- 当前公共区域采用单列内容流
- 表格容器使用白色卡片盒子包裹，顶部有模块标题“资产管理员”

### 公共内容区

- 外边距：顶部约 `20px`
- 内容卡片内边距：`10px`
- 卡片结构：
  - 模块标题
  - 数据表格

## 4. 核心组件

### 角色路由容器

- 页面本身承担“场景分发器”角色
- 设计上应允许不同角色工作台保持统一壳层，但内容完全不同
- 如果做成 Figma 页面，应体现“同一入口，不同角色视图”的设计关系

### 资产管理员表格

- 使用 `el-table`
- 三列结构：部门 / 岗位 / 人员
- 第一列支持纵向合并单元格，突出组织层级
- 人员字段以 `姓名(账号)` 形式串联展示

## 5. 交互规则

- 页面初始化先获取角色，再获取配置人员列表
- 角色切换是页面级逻辑，不建议用户手工在界面中切换
- 表格无复杂筛选、无批量操作，重点是“快速查阅配置关系”
- 合并单元格用于减少重复信息，提升部门维度可读性

## 6. 业务约束

- 角色枚举直接决定可见页面
- 该页是“后台入口页面”，所以比普通列表页更强调信息分流
- 资产管理员列表是组织配置视图，不是业务流水列表

## 7. 设计输出建议

- Figma 中建议拆成 2 层：
  - 角色入口页规范
  - 资产管理员表格页规范
- 若扩展此类页面，优先保持：
  - 单模块直入
  - 低干扰
  - 强组织结构感
